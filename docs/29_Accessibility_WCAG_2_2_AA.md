# Accessibility (WCAG 2.2 AA)

## Goals
The website must be usable by readers with visual, motor, cognitive, and assistive technology needs.

## Requirements
- Semantic HTML
- Correct heading hierarchy
- Accessible form labels and errors
- Keyboard operability for all controls
- Visible focus styles
- Contrast compliant dark theme
- Alt text for meaningful images
- Reduced motion support
- No color-only meaning
- Skip-to-content link

## Interactive map accessibility
- Keyboard navigation support
- Textual alternative representation
- Legend and node descriptions
- Focusable node selection
- Non-visual reading order view

## Forms accessibility
- Inline validation text
- Error summary on submit failure
- Clear required field indicators

## Content accessibility
- Plain-language summaries in addition to stylized copy
- Avoid text embedded in images for essential content
