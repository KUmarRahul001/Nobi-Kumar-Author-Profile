# AI Coding Instructions for Antigravity

## Objective
Implement the website exactly from the approved docs package without inventing extra product scope.

## Operating rules
- Build one feature at a time
- Prefer traceable requirement IDs
- Do not change routes without updating docs
- Keep components reusable and typed
- Keep design premium and restrained
- Do not add checkout in v1
- Do not add unnecessary libraries
- Preserve dark thriller identity
- Use content-first implementation

## Required implementation sequence
1. Scaffold routing and layout
2. Add content models
3. Build home and books pages
4. Add sample chapter flow
5. Add universe map
6. Add blog and comments
7. Add newsletter and contact forms
8. Add search and theme persistence
9. Add SEO, accessibility, and analytics
10. Add tests and deployment configuration

## Output expectations
- Production-ready code
- Clear commit-sized changes
- No placeholder TODOs in shipped code
- Minimal ambiguity
- High confidence validation

## Validation checklist
- All requirements traced
- All pages responsive
- Forms validated
- SEO metadata present
- Dark mode persistent
- Comments moderated
- Search works
