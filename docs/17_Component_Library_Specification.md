# Component Library Specification

## Core atoms
- Button
- Link
- Badge
- Icon
- Input
- Textarea
- Select
- Toggle
- Separator

## Core molecules
- Header
- Footer
- BookCard
- BlogCard
- SampleExcerptViewer
- SearchBar
- NewsletterForm
- ContactForm
- ThemeToggle
- Rating / review badge
- CTA strip

## Core organisms
- HeroSection
- FeaturedBookPanel
- UniverseMapPanel
- BlogFeedSection
- CommentThread
- RelatedBooksGrid
- FAQAccordion
- MediaKitBlock

## Reuse rules
- Buttons must be consistent across CTAs
- Book cards should share a single data schema
- Blog cards should share metadata rendering
- Forms should share validation and error display patterns

## Component quality bar
Each component must be:
- responsive
- accessible
- visually aligned to the brand
- testable
- reusable across pages
